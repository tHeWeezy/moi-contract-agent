# 产品需求文档 (PRD)：金盘 AI Portal

---

## 1. 项目背景 (Project Background)

### 1.1 产品定位

金盘 AI Portal 是一个面向 B 端企业内部用户的智能化信息门户系统，核心解决**项目进度追踪**、**客户档案管理**和**AI 辅助信息问询**三大业务场景。系统通过 AI 智能体对话界面作为统一入口，用户可通过自然语言查询项目或客户信息，并跳转至对应的详情页面进行深度查看。

### 1.2 目标用户

- 项目经理：追踪项目全生命周期进度（商务→技术→生产→物流→验收→结算）
- 销售代表：查询客户档案、合作历史、财务往来及风险信息
- 运营管理人员：通过矩阵视图总览多行项目的排产与货款进度
- 合同管理人员：通过合同智能体进行合同审查、信息提取等操作

### 1.3 系统架构概览

系统由三个独立子项目组成，通过页面跳转（`window.open`）实现互联：

| 子项目 | 技术栈 | 职责 |
|:---|:---|:---|
| `moi_c_all` | React 18 + TypeScript + Vite + Ant Design | 主入口：AI 对话门户 |
| `moi_c_ProjectInfo` | React 18 + TypeScript + Vite + Ant Design + react-router-dom | 项目进度系统（列表 + 矩阵 + 详情） |
| `moi_c_ClientInfo` | 原生 JS + Tailwind CSS (CDN) | 客户信息系统（列表 + 详情） |

部署方式：`moi_c_ProjectInfo` 和 `moi_c_ClientInfo` 的构建产物分别复制到 `moi_c_all/public/` 目录下，由 Vite 开发服务器统一托管。

## 2. 需求描述 (Requirement Description)

### 2.1 功能定义

#### 模块一：AI 对话门户 (`moi_c_all`)

##### F1.1 侧边栏导航 (Sidebar)

左侧固定侧边栏，包含以下导航项：

| 导航项 | 图标 | 说明 |
|:---|:---|:---|
| 对话 | `MessageOutlined` | 分类标题（非可点击项），右侧有编辑图标 |
| 通用智能体 | `RobotOutlined` | 通用 AI 对话入口 |
| 合同智能体 | `FileTextOutlined` | 当前激活项（高亮状态） |
| 问数智能体 | `BarChartOutlined` | 数据查询智能体入口 |
| 智能体广场 | `AppstoreOutlined` | 智能体市场/目录 |
| 消息 | `BellOutlined` | 消息通知入口 |

底部显示用户信息（头像 + 用户名 `wscs003`）及折叠按钮。

> **待确认**：侧边栏各导航项的点击跳转逻辑当前未实现，仅为静态展示。

##### F1.2 会话管理面板 (MiddlePanel)

位于侧边栏右侧的中间面板，功能包括：

- **智能体信息展示**：显示当前智能体名称（"合同智能体"）、描述（"审查并修订变更后的合同"）、头像及操作按钮（复制、剪贴）
- **创建新对话**：`+ 创建新对话` 按钮
- **搜索**：支持对话历史搜索
- **对话列表**：按日期分组展示历史对话记录（如 "3月16日 周一 → 新对话"、"1月12日 周一 → 321-信息提取 / 123-合同审查"）
- **底部功能入口**：`日志记录`、`智能跟催` 两个 Tab

> **待确认**：对话列表的切换、搜索过滤、日志记录和智能跟催功能当前为静态 UI，未实现交互逻辑。

##### F1.3 聊天对话区 (ChatPanel)

主聊天区域，核心交互流程：

1. **欢迎语**：首次进入显示 "你好，我是合同智能体，可以帮你审核合同、提取合同信息等等，帮你省时间、提效率~"
2. **用户输入**：底部输入框，支持多行输入（自动扩展，最大高度 120px），Enter 发送，Shift+Enter 换行
3. **输入框标签**：左上角显示 "信息问询 ∨" 标签（待确认：是否支持切换模式）
4. **消息流**：用户消息右对齐（蓝色气泡），助手消息左对齐（灰色气泡），自动滚动到最新消息
5. **加载状态**：助手回复期间显示加载指示器，输入框禁用

##### F1.4 关键词匹配查询引擎 (MockEngine)

当前为模拟实现，基于关键词匹配：

| 关键词 | 响应行为 |
|:---|:---|
| 包含 "项目" | 返回项目卡片（2 条 Mock 数据） |
| 包含 "客户" | 返回客户卡片（3 条 Mock 数据） |
| 其他 | 返回兜底提示："抱歉，测试阶段我仅支持通过包含'项目'或'客户'的关键词进行模拟查询，请重新输入。" |

响应延迟：800ms（模拟网络请求）。

> **待确认**：正式版本的 AI 对话引擎接入方案（LLM API / RAG 等）。

##### F1.5 数据卡片 (DataCard)

嵌入在助手消息气泡内的结构化数据展示组件：

- **项目卡片**：表格展示（所属项目编号、项目名称、当前状态、负责人），标题 "项目明细（共 N 份）"
- **客户卡片**：表格展示（客户编号、客户名称、联系人、客户级别），标题 "客户明细（共 N 份）"
- **表格滚动**：超过 5 行时启用纵向滚动（行高 47px × 5 = 235px）
- **导航按钮**：
  - 单条记录 → "查看项目详情" / "查看客户档案"（跳转详情页）
  - 多条记录 → "查看完整列表"（跳转列表页，携带搜索关键词）

##### F1.6 页面跳转逻辑 (Navigation)

所有跳转通过 `window.open(url, '_blank')` 在新标签页打开：

| 目标页面 | URL 格式 |
|:---|:---|
| 项目详情 | `./moi_c_ProjectInfo/index.html#/project/{projectId}` |
| 项目列表 | `./moi_c_ProjectInfo/index.html#/?keyword={keyword}` |
| 客户详情 | `./moi_c_ClientInfo/index.html?id={clientId}` |
| 客户列表 | `./moi_c_ClientInfo/list.html?keyword={keyword}` |

#### 模块二：项目进度系统 (`moi_c_ProjectInfo`)

##### F2.1 项目列表页 (ProjectListPage)

- **路由**：`#/`
- **展示形式**：卡片网格布局
- **卡片内容**：CRM 项目编码、工程名称、全局状态标签（正常/有风险/已延期/已完成）、是否出口项目标识
- **交互**：点击卡片跳转至项目进度矩阵页 `#/project/{projectId}`

##### F2.2 项目进度矩阵页 (ProjectProgressMatrixPage)

- **路由**：`#/project/:projectId`
- **页面结构**：

```
┌─────────────────────────────────────────────────┐
│ [Header] CRM编码 | 工程名称 | 全局状态 | ← 返回 │
├─────────────────────────────────────────────────┤
│ [商务卡片]     [生产卡片]      [物流卡片]        │
├─────────────────────────────────────────────────┤
│ [进度汇总面板] 排产/货款 统计概览                 │
├─────────────────────────────────────────────────┤
│ [排产进度矩阵] 行项目 × 节点1-13 表格            │
├─────────────────────────────────────────────────┤
│ [货款进度矩阵] 交货单 × 节点01-09 表格           │
└─────────────────────────────────────────────────┘
```

###### F2.2.1 顶部区域 (HeaderSection)

显示项目核心快照信息：
- CRM 项目编码
- 工程项目名称
- 全局状态标签（颜色编码）
- 返回按钮（回到项目列表）

###### F2.2.2 基础信息卡片 (BasicInfoCards)

分三类卡片展示项目基础字段：

| 卡片类型 | 包含字段 |
|:---|:---|
| 商务信息 | CRM项目编码、销售订单、合同不含税总金额、订单货币、最终客户名称、合同签订日期、产品组、事业部、创建日期、销售需求核发时间、销售代表处描述、销售代表描述、最终用户行业、售达方名称、是否投标、是否收到合同原件、信用冻结 |
| 生产信息 | 规格、生产工厂、站点 |
| 物流信息 | 送达方城市、总体交货状态 |

###### F2.2.3 进度汇总面板 (ProgressSummaryPanel)

展示两组汇总统计数据：

**排产进度汇总**：
- 行项目总数、已完成数、进行中数、延期数
- 风险预警文本
- 状态概括文本

**货款进度汇总**：
- 交货单总数、已完成数、进行中数、逾期数
- 风险预警文本
- 状态概括文本

###### F2.2.4 排产进度矩阵 (ProductionMatrixTable)

横向矩阵表格，行为行项目，列为节点 1-13：

- **行数据**：行项目编号、成品物料号、型号、订货数量
- **列数据**：节点 1（代表处发排产申请）至 节点 13（交货状态）
- **单元格**：显示节点状态图标（颜色编码），点击打开详情抽屉
- **默认显示**：5 行数据，超出部分滚动（表头固定）

**特殊功能**：
- **节点显示设置**：可通过弹窗控制节点列的显示/隐藏
- **自定义节点插入**：节点列之间 Hover 显示 `(+)` 按钮，点击弹出配置窗口添加自定义节点
- **配置模式**：可切换进入配置模式，逐单元格启用/禁用节点，退出时批量保存

###### F2.2.5 货款进度矩阵 (PaymentMatrixTable)

横向矩阵表格，行为交货单，列为货款节点 01-09：

| 矩阵编号 | 节点名称 | 原始节点编号 |
|:---|:---|:---|
| 01 | 预付款 | 2 |
| 02 | 发货款 | 8 |
| 03 | 到货款 | 14 |
| 04 | 安装调试款 | 15 |
| 05 | 通电验收款 | 16 |
| 06 | 竣工款 | 17 |
| 07 | 结算款 | 18 |
| 08 | 审计款 | 19 |
| 09 | 质保金 | 20 |

- **行数据**：交货单号、付款代码
- **单元格**：显示节点状态图标，点击打开详情抽屉
- **默认显示**：5 行数据，超出部分滚动（表头固定）

##### F2.3 详情抽屉 (DetailDrawer)

右侧滑出抽屉，根据节点类型展示不同内容：

| 节点范围 | 抽屉内容 |
|:---|:---|
| 节点 1 | 排产申请列表（排产申请表编号、要求完工时间、排产时间、排产方式、审批状态） |
| 节点 2 | 预付款信息（预付款截止日期、回款率） |
| 节点 3-5 | 通用节点信息（状态、时间、负责人、备注） |
| 节点 6 | 手动填报节点（确认交货日期输入） |
| 节点 7 | 嵌套下钻（Tabs 切换 7-1 设计 / 7-2 物料 / 7-3 生产） |
| 节点 8 | 发货款信息（发货款截止日期、回款率） |
| 节点 9-13 | 交货单列表（每个节点展示对应字段） |
| 节点 14-20 | 款项列表（每个节点展示对应截止日期 + 回款率） |

**节点 7 下钻详情**：

| 子节点 | 内容 |
|:---|:---|
| 7-1 设计进度 | 行项目级别的设计文档进度（计算单、BOM、高压图纸、低压图纸、铁芯图纸、外壳图纸等，每项含计划/实际日期对） |
| 7-2 物料进度 | 行项目级别的缺料情况（行项目缺料、生产工序缺料），异常项置顶排序 |
| 7-3 生产进度 | 行项目级别的工序进度（高压线圈绕制、低压线圈绕制、叠片、铁芯、装配、外壳、整装，每项含计划/实际日期） |

**回款率颜色编码**：

| 条件 | 颜色 | 说明 |
|:---|:---|:---|
| 回款率 = 100% | `#52C41A` 绿色 | 已完成 |
| 回款率 < 100% 且已到期 | `#FAAD14` 黄色 | 有风险 |
| 到期完全未回款 | `#F5222D` 红色 | 已延期 |
| 未回款但未到期 | `#1890FF` 蓝色 | 进行中 |

##### F2.4 节点状态体系

6 种节点状态，贯穿整个进度系统：

| 状态 | 标识符 | 颜色 | 说明 |
|:---|:---|:---|:---|
| 已完成 | `completed` | `#52C41A` 绿 | 流程通过 |
| 进行中 | `in_progress` | `#1890FF` 蓝 | 活跃节点 |
| 未开始 | `not_started` | `#D9D9D9` 灰 | 尚未启动 |
| 已延期 | `delayed` | `#F5222D` 红 | 超期未完成 |
| 有异常 | `abnormal` | `#FAAD14` 黄 | 逻辑预警 |
| 部分完成 | `partial` | `#722ED1` 紫 | 批量场景部分完成 |

##### F2.5 阶段配置

项目进度分为 6 个阶段，每阶段包含 3-4 个节点：

| 阶段 | 名称 | 包含节点 |
|:---|:---|:---|
| stage-1 | 商务阶段 | 节点 1-2（代表处发排产申请、预付款） |
| stage-2 | 技术阶段 | 节点 3-5（外部方案、内部方案、客户现场） |
| stage-3 | 生产阶段 | 节点 6-7（货期确认、完工） |
| stage-4 | 物流阶段 | 节点 8-13（发货款、代表处下发货单、项目经理下交货单、发货状态、过账状态、交货状态） |
| stage-5 | 验收阶段 | 节点 14-16（到货款、安装调试款、通电验收款） |
| stage-6 | 结算阶段 | 节点 17-20（竣工款、结算款、审计款、质保金） |

##### F2.6 出口项目配置

当 `isExport = true` 时，自动隐藏节点 1-5（商务阶段和技术阶段节点），仅显示节点 6-20。

## 3. 用户旅程与交互逻辑 (User Journey & Logic Flow)

### 3.1 核心用户旅程

#### 旅程一：通过 AI 对话查询项目进度

```
用户打开金盘 AI Portal
  → 侧边栏默认选中"合同智能体"
  → 聊天区显示欢迎语
  → 用户输入包含"项目"的问题（如"查一下项目进度"）
  → 系统返回项目数据卡片（表格形式）
  → 用户点击"查看项目详情"按钮
  → 新标签页打开项目进度矩阵页
  → 用户查看排产进度矩阵 / 货款进度矩阵
  → 点击某个单元格
  → 右侧抽屉弹出节点详情
  → 如为节点7，可通过 Tab 切换查看设计/物料/生产子节点
```

#### 旅程二：通过 AI 对话查询客户信息

```
用户在聊天区输入包含"客户"的问题（如"查一下客户信息"）
  → 系统返回客户数据卡片（表格形式）
  → 单条记录：点击"查看客户档案" → 新标签页打开客户详情页
  → 多条记录：点击"查看完整列表" → 新标签页打开客户列表页
  → 客户列表页：搜索/筛选 → 点击卡片 → 进入客户详情页
  → 客户详情页：查看工商背景、业务合作、评级、财务往来、账龄明细、风险记录
```

#### 旅程三：直接访问项目列表

```
用户直接访问项目列表页（或从 AI 对话跳转）
  → 查看项目卡片网格
  → 点击某个项目卡片
  → 进入项目进度矩阵页
  → 查看基础信息卡片（商务/生产/物流）
  → 查看进度汇总面板
  → 在排产进度矩阵中点击单元格查看节点详情
  → 在货款进度矩阵中点击单元格查看款项详情（含回款率颜色编码）
```

#### 旅程四：项目进度矩阵高级操作

```
进入项目进度矩阵页
  → 点击"节点显示设置" → 弹窗中勾选/取消节点 → 控制矩阵列的显示隐藏
  → 在节点列之间 Hover → 出现 (+) 按钮 → 点击添加自定义节点
  → 点击"配置模式"开关 → 进入配置模式
  → 点击单元格切换节点的启用/禁用状态
  → 退出配置模式 → 弹出确认对话框 → 保存更改
```
